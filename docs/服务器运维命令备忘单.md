# 服务器运维命令备忘单

这份文档记录生产服务器上最常用的三类操作：更新代码、重新启动、服务器重启后的自动拉起。

## 1. 更新代码

前提是服务器能够访问 GitHub。进入项目目录后直接拉取主分支最新代码：

```bash
cd /bussiness/bussinessV3
git pull origin main
```

如果仓库代码有更新，拉取完成后建议重建服务镜像：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
```

如果 `git pull` 报网络错误，先不要反复重试，说明服务器出网或代理配置有问题。先修复网络，再同步代码。

## 2. 重新启动

如果只是想让当前服务重新起一次，优先用下面这条：

```bash
cd /bussiness/bussinessV3
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
```

这条命令会：

- 重新构建 `api` 和 `web`
- 拉起 `db`、`api`、`web`
- 保持现有数据库卷不变

如果只是容器卡住、配置没变，也可以只重启容器：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml restart
```

常用检查命令：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml ps
docker compose --env-file .env.production -f docker-compose.prod.yml logs -f
```

## 3. 服务器重启后自动启动

这个项目已经在 `docker-compose.prod.yml` 里给三个服务都加了：

```yaml
restart: unless-stopped
```

含义是：

- Docker 容器会在异常退出后自动重启
- 服务器重启后，只要 Docker 服务本身正常启动，容器也会跟着自动拉起
- 一般不需要人工再执行一次 `up`

但要注意两点：

1. 服务器的 Docker 服务本身要开机自启。
2. 容器必须曾经被成功创建过。

可以在服务器上检查 Docker 服务：

```bash
systemctl status docker
```

如果需要让 Docker 开机自启：

```bash
systemctl enable docker
```

如果 Docker 没有启动，可以手工启动：

```bash
systemctl start docker
```

## 4. 推荐日常流程

最常见的运维流程就是下面这三步：

```bash
cd /bussiness/bussinessV3
git pull origin main
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --build
docker compose --env-file .env.production -f docker-compose.prod.yml ps
```

## 5. 备注

- 生产环境配置文件是 `.env.production`
- 数据库数据保存在 Docker volume 里，重新 `up` 不会自动清空
- 如果要彻底重装数据库，需要额外执行 `down -v`
- 如果服务器无法直连 GitHub，必须先解决出网或代理问题，`git pull` 才能成功
