## 上线前必须完成

1. 准备服务器
- Linux 服务器
- Docker Engine
- Docker Compose Plugin
- 开放 `80` 端口

2. 准备域名和邮件服务
- 网站访问可以暂时先用 IP 测试
- 真实验证邮件和重置密码邮件必须使用域名邮件方案
- 建议方案：`Resend + 你自己的域名`
- 至少准备：
  - 一个可控域名
  - 一个发信子域名，例如 `mail.your-domain.com`
  - Resend API Key 或其他 SMTP 凭据

3. 配置生产环境变量
- 基础变量：
  - `POSTGRES_PASSWORD`
  - `JWT_SECRET`
  - `DEFAULT_TEACHER_PASSWORD`
  - `AI_API_KEY`
  - `WEB_PORT`
  - `CONTAINER_REGISTRY_MIRROR`（国内服务器可用，例如 `docker.m.daocloud.io/library/`）
  - `APT_MIRROR`（国内服务器可用，例如 `http://mirrors.aliyun.com/debian`）
- 邮件相关变量：
  - `APP_BASE_URL`
  - `EMAIL_VERIFICATION_REQUIRED`
  - `EMAIL_VERIFICATION_EXPIRES_HOURS`
  - `PASSWORD_RESET_EXPIRES_MINUTES`
  - `MAIL_MODE=smtp`
  - `SMTP_HOST`
  - `SMTP_PORT`
  - `SMTP_SECURE`
  - `SMTP_USER`
  - `SMTP_PASS`
  - `SMTP_FROM`

4. 确认部署版本
- 不建议直接长期跟 `main`
- 建议先提交当前改动，再打 tag 部署

## 当前发现的上线风险

### 高优先级

- `docker-compose.prod.yml` 原先没有把 `APP_BASE_URL`、`MAIL_MODE`、`SMTP_*` 传给 API 容器
  - 已在当前工作区修复
  - 如果不修，线上邮件功能无法工作

### 中优先级

- 当前部分旧文档存在中文乱码
  - 不影响运行
  - 但会影响交接和上线操作体验
- 当前邮件真实投递仍依赖外部 SMTP 供应商配置完成
  - 代码已支持
  - 但没有域名和 SMTP 凭据就不能正式发信
- 试用期如需降低注册门槛，可临时设置 `EMAIL_VERIFICATION_REQUIRED=false`
  - 学生注册后可直接登录
  - 后续正式上线再改回 `true`

### 低优先级

- 还缺自动化 smoke test
  - 当前主要依赖人工验证
- 还没有 HTTPS 自动化配置
  - 建议由外层反向代理或云厂商 LB 终止 TLS

## 推荐上线顺序

1. 提交并推送当前改动
2. 准备 `.env.production`
3. 在服务器上拉取固定版本
4. 执行：

```bash
npm run prod:up
npm run prod:migrate
npm run prod:seed
npm run prod:seed:content
```

5. 检查：

```bash
curl http://127.0.0.1:${WEB_PORT:-80}
curl http://127.0.0.1:${WEB_PORT:-80}/api/health
curl http://127.0.0.1:${WEB_PORT:-80}/api/health/db
```

6. 人工回归：
- 教师登录
- 学生注册
- 邮箱验证
- 学生登录
- 忘记密码
- 重置密码
- 模拟谈判发送一轮消息

## 推荐的 Resend SMTP 配置

```env
APP_BASE_URL=https://your-domain.com
MAIL_MODE=smtp
SMTP_HOST=smtp.resend.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=resend
SMTP_PASS=your_resend_api_key
SMTP_FROM=BIZ Chinese <no-reply@mail.your-domain.com>
EMAIL_VERIFICATION_EXPIRES_HOURS=24
PASSWORD_RESET_EXPIRES_MINUTES=60
```

如果使用 `587` 端口，则改为：

```env
SMTP_PORT=587
SMTP_SECURE=false
```

## 国内服务器镜像源建议

如果服务器拉取 Docker Hub 或 Debian apt 源很慢，可在 `.env.production` 配置：

```env
CONTAINER_REGISTRY_MIRROR=docker.m.daocloud.io/library/
APT_MIRROR=http://mirrors.aliyun.com/debian
```
