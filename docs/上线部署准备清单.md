## 服务器上线前必查

### 1. 服务器基础环境

- Ubuntu / Debian Linux
- Docker Engine
- Docker Compose Plugin
- 已放通对外访问端口，例如 `80`

### 2. 目录位置

建议统一使用：

```bash
/bussiness/bussinessV3
```

### 3. 生产环境变量

确认 `.env.production` 已存在并至少填写：

- `POSTGRES_PASSWORD`
- `JWT_SECRET`
- `DEFAULT_TEACHER_USERNAME` / `DEFAULT_TEACHER_PASSWORD`，它们只是 `prod:seed` 的输入值，不是数据库里预置的固定账号
- `DEFAULT_ADMIN_USERNAME` / `DEFAULT_ADMIN_PASSWORD`，它们只是初始化输入值，系统管理员会写入数据库并同时拥有 `admin` 和 `teacher` 角色
- `APP_BASE_URL`
- `WEB_PORT`
- `AI_API_KEY` 或接受 AI 降级

初始化顺序是：

1. `npm run prod:up`
2. `npm run prod:migrate`
3. `npm run prod:seed`
4. `npm run prod:seed:content`

### 4. 数据初始化命令

首次部署或新库必须执行：

```bash
npm run prod:migrate
npm run prod:seed
npm run prod:seed:content
```

## 日常发布顺序

### 本地

```powershell
git add .
git commit -m "your message"
git push origin main
```

### 服务器

```bash
cd /bussiness/bussinessV3
git pull origin main
npm run prod:up
```

如有数据库变更：

```bash
npm run prod:migrate
```

如有种子内容变更：

```bash
npm run prod:seed
npm run prod:seed:content
```

## 服务器重启恢复

确认 Docker 开机自启：

```bash
sudo systemctl enable docker
sudo systemctl start docker
```

如果重启后服务没有自动起来：

```bash
cd /bussiness/bussinessV3
npm run prod:up
```

## 上线后检查

查看容器：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml ps
```

查看 API 日志：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml logs -f api
```

查看健康接口：

```bash
curl http://127.0.0.1:${WEB_PORT:-80}/api/health
curl http://127.0.0.1:${WEB_PORT:-80}/api/health/db
```

## 风险提醒

- `JWT_SECRET` 为空时，API 无法启动。
- `docker compose down -v` 会清空数据库数据。
- `MAIL_MODE=preview` 不适合正式环境邮件发送。
- 如果 AI 配置为空，系统会降级，不会阻止基础业务流程启动。
