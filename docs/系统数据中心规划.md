# 系统数据中心规划

Last updated: 2026-04-12

## 目标

系统数据中心是教师/管理员后台里的只读运维与教学管理入口，目标是同时服务两类场景：

- 开发调试：快速查看当前数据库各表记录、AI 调用结果、学习事件和异常状态。
- 管理查看：用业务视角理解学生、内容、实训、AI 的当前状态，而不是直接面对一堆孤立表。

第一阶段坚持只读，避免误删、误改教学数据。编辑、禁用、导出等能力后续按权限逐项开放。

## 信息架构

页面放在教师后台 `2.5 系统数据`。

左侧按数据域组织表：

- 用户与权限：用户、角色、用户角色
- 学生档案：学生资料、学生登录身份
- 学习内容：业务阶段、阶段任务、学习资源、AI 场景提示词
- AI 与实训：实训会话、实训消息、AI 调用日志、消息分析结果
- 行为记录：练习事件、学习快照
- 旧版兼容：旧版会话、旧版消息

主区域分三层：

- 系统总览：关键数量、AI 状态、今日活动、最近错误。
- 数据列表：按表查看记录，支持搜索、状态筛选、时间范围、分页。
- 记录详情：点击行后查看完整 JSON，后续扩展关联记录入口。

## 后端设计

后端入口为 `apps/api/src/routes/admin.ts`，挂载在 `/api/admin`。

所有接口必须经过 `requireAuth` 和教师/管理员角色校验。数据表访问必须走白名单 registry，不允许前端传任意 Prisma model 名称。

当前接口：

- `GET /api/admin/tables`
  - 返回白名单表元数据。
  - 元数据包括分组、搜索字段、摘要列、状态字段、时间字段。
- `GET /api/admin/tables/:tableKey`
  - 返回指定表分页数据。
  - 支持 `search`、`statusField`、`status`、`dateField`、`dateRange`。
  - 隐藏敏感字段，例如 `users.passwordHash`。
- `GET /api/admin/overview`
  - 返回系统总览指标。
  - 包括用户、学生、内容、实训、AI 调用、fallback、最近错误等。

## 前端设计

前端入口为 `apps/web/components/TeacherDashboard.tsx`。

当前页面应提供：

- 顶部总览卡片：用户、学生、今日消息、AI 调用、fallback、资源、场景、最近错误。
- 左侧表导航：按数据域折叠或分组展示。
- 表格工具栏：搜索、状态筛选、时间范围筛选、刷新。
- 表格列表：优先显示 registry 提供的 `summaryColumns`，长文本截断。
- 记录详情：点击任意行查看完整 JSON。
