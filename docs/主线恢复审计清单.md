## 关键观察

### 1. `chore/fix-startup-flow` 理论上可以作为第一恢复批次

因为它是基于当前 `main` 的 2 个新增提交，不是历史分叉。  
从 Git 结构上看，优先恢复这层最合理。

### 2. `stash@{0}` 不适合直接整体 apply

原因：

- 它不是正式提交。
- 它叠加在 `chore/fix-startup-flow` 之上。
- 它包含大体量 UI 改造和依赖升级。
- 它还动了根目录旧 `prisma/schema.prisma`，容易把遗留噪音重新带回主线。

### 3. 当前 `main` 缺少系统数据中心的真实后端入口

已确认当前 `main` 中：

- 不存在 `apps/api/src/routes/admin.ts`
- 不存在 `apps/web/components/ResourcePanel.tsx`

这说明“系统数据中心”和“内容数据库前端接线”都没有真正进入当前主线。
