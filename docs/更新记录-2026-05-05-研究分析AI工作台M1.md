# 更新记录：研究分析 AI 工作台 M1（自然语言查库）

更新时间：2026-05-05

## 本次目标

实现研究分析工作台 M1：研究者可通过自然语言提问，由大模型生成只读 SQL 查询数据库并返回分析结论。

## 后端新增

- 新增路由：`POST /api/research/ai/query`
- 文件：`apps/api/src/routes/researchAi.ts`
- 挂载入口：`apps/api/src/index.ts`

### 路由能力

1. 校验请求参数（`question`）
2. 权限校验（仅 `admin/teacher`）
3. 调用模型将问题转换为 SQL（仅允许 SELECT）
4. SQL 安全校验：
   - 禁止多语句
   - 禁止 DML/DDL（insert/update/delete/drop 等）
   - 仅允许白名单数据表
5. 执行查询并返回行数据
6. 调用模型对结果生成简要中文洞察

## Token / 模型配置说明

大模型 Token 已有可配置入口：

- 环境变量：
  - `AI_API_KEY`（或 `DEEPSEEK_API_KEY`）
  - `AI_PROVIDER`
  - `AI_BASE_URL`
  - `AI_MODEL`
  - `AI_TIMEOUT_MS`
- 管理后台“系统管理”页面也可在线配置 AI 参数并保存。

此外，本次 M1 新增研究查询相关环境变量：

- `RESEARCH_AI_ALLOWED_TABLES`：允许查询的数据表白名单（逗号分隔）

示例：

```bash
RESEARCH_AI_ALLOWED_TABLES=teaching_groups,teaching_group_members,users,student_profile,practice_events
```

## 使用方式（接口）

```bash
curl -X POST http://localhost:8000/api/research/ai/query \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"question":"最近30天各分组活跃趋势"}'
```


## 前端集成（Teacher Dashboard）

- 在 `研究分析工作台` 页面增加 **AI SQL Copilot（M1）** 区块。
- 支持输入自然语言问题，调用 `/api/research/ai/query`，展示：
  - AI 分析结论
  - 生成 SQL
  - 返回行数与耗时

## 管理员如何配置大模型 Token

可通过两种方式配置：

1. 环境变量配置（部署层）
2. 系统管理页面在线配置（推荐运维使用）

系统管理页面路径：教师端进入“系统管理”页面，填写并保存 AI Provider、Base URL、Model、API Key、Timeout。


## M1.1 增强（交互与数据导出）

- 增加提问模板快捷按钮（趋势、分组人数、事件趋势）。
- 增加结果表格分页预览（每页 10 条，前后翻页）。
- 增加 CSV 导出按钮，研究者可下载本次查询结果进行二次分析。


## M1.2 增强（查询效率）

- 新增 SQL 一键复制按钮。
- 新增查询历史（本地保存最近 20 条，快捷回填问题）。
- 新增结果表格筛选（关键字包含匹配）。
- 新增结果表格列排序（指定列 + 升降序切换）。

## M2（图表建议 + 多轮追问最小可用）

- 后端 `POST /api/research/ai/query` 新增返回字段：
  - `chartSuggestion`：`line | bar | table`
  - `followupPrompts`：2~3 条推荐追问
- 推荐值由模型基于问题、SQL 与结果样例生成，并在服务端做结构校验与兜底（解析失败时回退 `table`）。
- 前端研究工作台新增“图表建议”展示。
- 前端新增“推荐追问”按钮，点击可一键回填到问题输入框，便于进入多轮追问。
- 保持原有 M1.2 功能不变（SQL 复制、历史、筛选、排序、分页、CSV）。
