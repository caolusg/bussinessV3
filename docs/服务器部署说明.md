# 服务器部署说明

Last updated: 2026-05-03

## 部署目标

当前项目的生产部署方式是：

- `db`: PostgreSQL
- `api`: Express + Prisma
- `web`: React/Vite 构建产物，由 nginx 提供静态访问并代理 `/api`

生产环境使用：

- `docker-compose.prod.yml`
- `.env.production`

## 服务器首次部署

建议目录：

```bash
/bussiness/bussinessV3
```

### 1. 拉取代码

```bash
cd /bussiness
git clone https://github.com/caolusg/bussinessV3.git
cd bussinessV3
git checkout main
```

### 2. 准备生产环境变量

```bash
cp .env.production.example .env.production
```

至少确认这些变量：

```env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=请改成强密码
POSTGRES_DB=bussinessv3

JWT_SECRET=请改成长随机字符串
JWT_EXPIRES_IN=8h
VITE_AUTH_TOKEN_STORAGE=session
VITE_AUTH_IDLE_TIMEOUT_MINUTES=120
VITE_AUTH_ABSOLUTE_TIMEOUT_HOURS=8
BCRYPT_ROUNDS=10

DEFAULT_TEACHER_USERNAME=teacher
DEFAULT_TEACHER_PASSWORD=请改成正式密码

APP_BASE_URL=http://你的域名或IP
WEB_PORT=80
CORS_ORIGINS=

MAIL_MODE=preview
SMTP_HOST=
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=
SMTP_PASS=
SMTP_FROM=

AI_ENABLED=true
AI_PROVIDER=deepseek
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
AI_API_KEY=
AI_PROXY_URL=
AI_TIMEOUT_MS=15000
```

说明：

- `JWT_SECRET` 不能为空，否则 API 无法启动。
- `DEFAULT_TEACHER_USERNAME` / `DEFAULT_TEACHER_PASSWORD` 只是 seed 的输入值，真正的教师账号会在执行 `npm run prod:seed` 后写入数据库。
- `DEFAULT_TEACHER_PASSWORD` 建议首次部署后立即修改。
- `MAIL_MODE=preview` 仅适合测试，正式环境建议改为 `smtp`。
- 如果不需要真实 AI，可先保留 `AI_API_KEY=` 空值，系统会走降级逻辑。

### 3. 启动生产服务

```bash
npm run prod:up
```

### 4. 初始化数据库

首次部署时执行：

```bash
npm run prod:migrate
npm run prod:seed
npm run prod:seed:content
```

## 日常更新

当本地代码已经提交并推送到 GitHub 后，服务器更新流程：

```bash
cd /bussiness/bussinessV3
git pull origin main
npm run prod:up
```

如果数据库结构有变更，再补：

```bash
npm run prod:migrate
```

如果基础数据或内容种子有更新，再补：

```bash
npm run prod:seed
npm run prod:seed:content
```

## 服务器重启后如何恢复

当前生产 compose 已配置 `restart: unless-stopped`，理论上服务器重启后会自动恢复。

你只需要保证 Docker 开机自启：

```bash
sudo systemctl enable docker
sudo systemctl start docker
```

如果重启后服务没有自动恢复，手动执行：

```bash
cd /bussiness/bussinessV3
npm run prod:up
```

## 状态检查

查看容器状态：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml ps
```

查看日志：

```bash
npm run prod:logs
```

只看 API：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml logs -f api
```

只看数据库：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml logs -f db
```

## 健康检查

检查前端：

```bash
curl http://127.0.0.1:${WEB_PORT:-80}
```

检查 API：

```bash
curl http://127.0.0.1:${WEB_PORT:-80}/api/health
curl http://127.0.0.1:${WEB_PORT:-80}/api/health/db
```

## 数据持久化

生产数据库数据保存在 Docker volume：

```text
pgdata_prod
```

注意：

- `docker compose down` 不会默认删除数据。
- `docker compose down -v` 会删除 volume，数据库数据会丢失。

## 备份建议

导出数据库：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml exec -T db \
  pg_dump -U "$POSTGRES_USER" "$POSTGRES_DB" > backup.sql
```

恢复数据库：

```bash
cat backup.sql | docker compose --env-file .env.production -f docker-compose.prod.yml exec -T db \
  psql -U "$POSTGRES_USER" "$POSTGRES_DB"
```
