## 本次更新内容

本次更新增加了教师后台的两个实际功能模块：

- `2.1 教学资源管理`：教师可以按业务阶段管理学习资源，包括词汇、表达句型、商务知识；支持新增、编辑、停用、搜索。
- `2.2 分组管理`：教师可以新建教学分组、编辑分组、停用分组、把学生加入或移出分组，并查看 HSK、专业、国籍统计。

## 数据库变化

本次更新包含数据库迁移，新增两张表：

- `teaching_groups`
- `teaching_group_members`

迁移文件：

```text
apps/api/prisma/migrations/20260504152000_teaching_groups/migration.sql
```

因此服务器 `git pull` 以后，不能只重启前端，必须执行 Prisma migration。

## 服务器生效命令

等服务器可以正常连接 GitHub 后，在服务器执行：

```bash
cd /bussiness/bussinessV3
git pull origin main
```

拉取成功后执行：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml build api web
docker compose --env-file .env.production -f docker-compose.prod.yml run --rm api npm run prisma:migrate:deploy
docker compose --env-file .env.production -f docker-compose.prod.yml up -d api web
docker compose --env-file .env.production -f docker-compose.prod.yml ps
```

## 如果只是想一次性执行

确认已经 `git pull` 成功后，可以直接复制：

```bash
cd /bussiness/bussinessV3
docker compose --env-file .env.production -f docker-compose.prod.yml build api web
```

## 注意事项

- 不要执行 `docker compose down -v`，否则会删除数据库 volume，已有学生、教师、会话数据会丢失。
- `prisma:migrate:deploy` 是安全的增量迁移命令，已经执行过的迁移不会重复执行。
- 如果 `git pull` 仍然失败，先解决服务器访问 GitHub 的网络问题；在没有拉到新代码前，执行迁移和重建不会得到本次新功能。
